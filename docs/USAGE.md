## 概述

vSphere Nexus 是一个基于 Web 的 vSphere 虚拟机批量管理工具。它通过 `ovftool` 和 vSphere SOAP API 与 ESXi/vCenter 交互，提供以下功能：

- **批量部署**：从 vSphere 模板批量创建虚拟机，支持并发执行
- **虚拟机概览**：查看所有 VM 的电源状态、配置、IP、操作系统等信息
- **批量电源控制**：一键开机、关机、重启多台虚拟机
- **批量销毁**：批量关机并删除虚拟机
- **实时日志**：通过 SSE 实时查看部署和操作日志
- **失败重试**：部署失败后仅重试失败的 VM
- **导出 CSV**：导出虚拟机清单为 CSV 文件

### 核心限制

- **部署源必须是 vSphere 模板**，不支持本地上传 OVA 文件
- 如果只有 OVA 文件，需要先手动导入到 vSphere 并转换为模板

## 快速开始

1. 打开 vSphere Nexus Web 界面（默认 http://localhost:4173）
2. 如果启用了认证，先输入用户名和密码登录
3. 在顶部连接面板填写 vSphere 地址、用户名、密码，点击 **"连接"**
4. 通过顶部 Tab 切换功能：
   - **批量部署**：选择模板、配置参数、生成 VM 清单、开始部署
   - **虚拟机概览**：查看所有 VM 状态、批量操作（开机/关机/重启/删除）、导出清单

## 连接 vSphere

所有功能共用顶部的连接面板。

### 操作步骤

1. **选择平台类型**：
   - **ESXi**：直连单台主机，适合小环境
   - **vCenter**：连接 vCenter Server，可管理多数据中心和集群
2. **填写连接信息**：
   - 地址：ESXi 或 vCenter 的 IP 或域名
   - 用户名：ESXi 默认 `root`，vCenter 为域用户
   - 密码：vSphere 登录密码
3. **点击 "连接"**

连接成功后会显示绿色标签和成功消息，系统自动读取 vSphere 资源。

### 连接后的自动发现

连接成功后自动读取以下资源：

| 资源 | 说明 |
|------|------|
| 部署目标 | vCenter 模式下的计算资源/集群 |
| Datastore | 存储列表，显示可用空间 |
| VM Folder | 虚拟机文件夹（可选） |
| 网络 | 可用端口组和分布式端口组 |
| 模板 | vSphere 中的虚拟机模板 |
| VM 列表 | 所有普通虚拟机（用于概览和清理） |

### 注意事项

- 修改任何连接字段会自动断开当前连接，需要重新点击 "连接"
- 会话缓存 10 分钟，重复操作无需重新登录 vSphere
- vSphere 连接使用 SOAP API（端口 443），确保网络可达

## 批量部署

### 配置部署参数

连接成功后，切换到 **"批量部署"** Tab，配置以下参数：

#### 部署目标

| 参数 | 说明 |
|------|------|
| 部署目标 | vCenter 模式下选择计算资源/集群 |
| Datastore | 目标存储，下拉显示可用空间 |
| VM Folder | 虚拟机存放的文件夹（可选） |
| 磁盘模式 | `thin`（精简）、`thick`（厚置备）、`eagerZeroedThick`（厚置备快速归零） |
| 部署后开机 | 勾选后部署完成自动开机 |

#### 部署源

从下拉列表中选择一个 vSphere 模板。只有模板类型的虚拟机会出现在列表中。

选择模板后：
- 模板中的网络适配器自动填充到网络映射
- 系统自动选择匹配的 Datastore 和网络

#### 网络映射

将模板中的源网络映射到目标端口组：

| 列 | 说明 |
|----|------|
| 模板源网络 | 从模板中自动读取的网络适配器 |
| 目标 Port Group | 目标环境中的端口组（下拉选择） |

点击 "+" 添加更多映射，点击垃圾桶图标删除映射。

#### 并发数

控制同时运行的 ovftool 进程数量（1-10）：

| 场景 | 建议值 |
|------|--------|
| 本地网络 / SSD 存储 | 3-5 |
| 低带宽 / 机械硬盘 | 1-2 |
| 高性能存储 / 万兆网络 | 5-10 |

### 生成虚拟机清单

#### 简单生成模式

适合规则命名的批量部署：

| 参数 | 示例 | 生成的名称 |
|------|------|------------|
| 前缀 | `web` | - |
| 数量 | `3` | - |
| 起始编号 | `1` | - |
| 编号位数 | `2` | `web-01`、`web-02`、`web-03` |

更多示例：

| 前缀 | 数量 | 起始 | 位数 | 结果 |
|------|------|------|------|------|
| `db` | 5 | 1 | 3 | `db-001` ~ `db-005` |
| `node` | 10 | 0 | 2 | `node-00` ~ `node-09` |
| `test` | 1 | 1 | 1 | `test-1` |

#### 手动清单模式

适合不规则命名或需要精确控制名称的场景：

- 在文本框中一行输入一个名称
- 支持 `{{index}}` 变量

### 提交部署

#### 干跑模式（默认开启）

点击 **"生成部署预览"**，系统会：

1. 生成所有 VM 的 ovftool 命令（密码脱敏显示）
2. 记录到任务面板
3. **不会实际执行部署**

建议每次正式部署前先干跑一次，确认命令和参数正确。

#### 提交前自动检查

提交时系统自动检查：

1. **VM 名称冲突**：如果目标环境中已有同名 VM，弹出警告
   - 点击 "返回修改" 修改名称
   - 点击 "仍然部署" 强制提交
2. **Datastore 容量**：根据模板大小和部署数量估算所需空间

#### 正式部署

1. 关闭 "干跑模式" 开关（Tab 栏右侧）
2. 点击 **"开始批量部署"**
3. 右侧任务面板实时显示进度和日志

## 虚拟机概览

切换到 **"虚拟机概览"** Tab，查看当前环境中所有虚拟机的详细信息。

### 统计栏

顶部显示 6 个统计指标：

| 指标 | 说明 |
|------|------|
| 总 VM | 所有普通虚拟机数量 |
| 运行中 | 电源状态为 poweredOn 的数量 |
| 已关机 | 电源状态为 poweredOff 的数量 |
| 总 CPU | 所有 VM 的 CPU 核数之和 |
| 总内存 | 所有 VM 的内存之和（GB） |
| 总存储 | 所有 VM 的已提交存储之和 |

### 搜索和筛选

- **搜索框**：按名称、IP 地址、数据中心、操作系统模糊搜索
- **状态筛选**：全部 / 运行中 / 已关机

### VM 列表

| 列 | 说明 | 可排序 |
|----|------|--------|
| ☐ | 复选框，用于批量操作 | - |
| 名称 | VM 名称 | ✓ |
| 状态 | 运行中（绿色）/ 已关机（红色）/ 已挂起 | ✓ |
| IP 地址 | VMware Tools 上报的 IP 地址 | ✓ |
| CPU | 核数 | ✓ |
| 内存 | MB 或 GB | ✓ |
| 存储 | 已提交存储量 | ✓ |
| 操作系统 | Guest OS 全名 | - |
| 创建时间 | VM 创建时间 | ✓ |

点击列标题排序，再次点击切换升序/降序。

> **注意**：IP 地址需要 VMware Tools 运行才能获取，未安装 VMware Tools 的 VM 显示为 "-"。

### 批量操作

1. 勾选要操作的 VM（点击行左侧复选框，或表头全选）
2. 出现蓝色操作栏：**已选 N 台**
3. 点击操作按钮：
   - **开机**：启动已关机的 VM（已是运行状态的自动跳过）
   - **关机**：强制关机（已关机的自动跳过）
   - **重启**：强制重启（仅对运行中的 VM 执行）
   - **关机并删除**（红色）：强制关机后删除 VM，**此操作不可撤销**
4. 确认弹窗后执行，右侧任务面板显示进度

### 刷新数据

点击标题栏右侧的刷新图标，重新连接 vSphere 获取最新数据。

### 导出 CSV

点击 **"导出 CSV"** 按钮，将当前筛选后的 VM 列表导出为 CSV 文件。文件包含 BOM 头，Excel 可直接打开无乱码。

导出内容：名称、状态、IP 地址、CPU 核数、内存(MB)、存储(GB)、操作系统、数据中心、创建时间。

## 任务管理

### 任务面板

右侧任务面板始终显示当前选中任务的信息：

- **状态条**：任务状态 + 完成进度（如 3/10）
- **进度条**：可视化进度
- **进度统计**：N 完成 / N 失败 / N 待执行
- **命令预览**：最多显示前 5 条 ovftool 命令（密码已脱敏）
- **实时日志**：通过 SSE 推送，自动滚动到最新
- **导出按钮**：将完整日志导出为 `.log` 文件

### 任务列表

左侧边栏显示所有历史任务，按创建时间倒序排列。点击任务名切换查看。

### 任务状态

| 状态 | 含义 |
|------|------|
| `queued` | 排队中 |
| `running` | 正在执行 |
| `succeeded` | 全部成功 |
| `failed` | 部分或全部失败 |
| `cancelled` | 用户取消 |
| `interrupted` | 服务重启导致中断 |

### 任务类型

| 图标 | 类型 | 说明 |
|------|------|------|
| ▶ | 部署 | 批量部署虚拟机 |
| ⏻ | 电源 | 批量开机/关机/重启 |
| 🗑 | 销毁 | 批量关机并删除 |

### 取消任务

任务运行中时，任务面板右上角出现取消按钮（X 图标）。点击后中止操作。

### 重试失败项

部署任务状态为 `failed` 时，任务面板出现重试按钮（循环箭头图标）。点击后仅重新部署失败的 VM，成功的不会重复。

## 认证登录

如果服务端配置了 `NEXUS_USER` 和 `NEXUS_PASS` 环境变量，访问 Web 界面时会显示登录页面。

| 特性 | 说明 |
|------|------|
| 登录方式 | 用户名 + 密码 |
| Token 有效期 | 24 小时 |
| Token 存储 | 浏览器 localStorage |
| 限速保护 | 同一 IP 15 分钟内最多 10 次登录尝试 |
| 会话过期 | 自动跳转到登录页 |

不设置这两个变量则不启用认证（适合内网可信环境）。

## 数据安全

| 安全措施 | 说明 |
|----------|------|
| 密码存储 | vSphere 密码仅在浏览器 `sessionStorage` 中暂存，关闭标签页即清除 |
| 日志脱敏 | 任务日志中 ovftool 命令的密码只显示首字符 + 星号 |
| 服务端加密 | `payloads.json` 中的密码使用 AES-256-GCM 加密，密钥在 `data/.payload-key` |
| SSE 认证 | 实时日志通过短期 ticket（30 秒有效）认证，避免 token 泄漏 |
| TLS | 服务端与 vSphere 通信不验证证书（支持自签名证书），建议前端通过 Nginx 加 HTTPS |

## 常见问题

### 部署失败：Already exists

目标环境中已有同名 VM。解决方式：
1. 在 vSphere Client 中删除或重命名冲突的 VM
2. 在 vSphere Nexus 中修改 VM 名称

### 连接成功但看不到模板

确认目标环境中存在模板类型的虚拟机。普通 VM 不会出现在模板列表。在 vSphere Client 中右键虚拟机 → "转换为模板"。

### 网络映射中没有源网络

模板没有配置网络适配器。在 vSphere Client 中编辑模板设置，添加网络适配器。

### 概览中 IP 地址显示为 "-"

VM 未安装 VMware Tools 或 VMware Tools 未运行。安装并启动 VMware Tools 后，vSphere 会自动上报 IP 地址。

### 批量电源操作后概览数据没变

点击概览面板标题栏的刷新图标重新获取最新数据。

### 部署速度慢

1. 调高并发数（建议 3-5）
2. 确认网络带宽是否充足
3. 确认 Datastore 的 IO 性能
4. 使用 `thin` 磁盘模式比 `thick` 更快

### 重启服务后任务变成 "interrupted"

正常行为。服务重启时正在执行的任务无法恢复。可以查看日志确认哪些 VM 已成功，使用重试功能重新执行失败项。

### 导出的 CSV 在 Excel 中乱码

CSV 文件已包含 UTF-8 BOM 头，Excel 2016+ 可正常打开。旧版本 Excel 请通过 "数据" → "从文本/CSV" 导入，选择 UTF-8 编码。
